PERSONAL AGENT PLATFORM

tsumugi — a personal multi-repo AI agent platform

A 5-repo personal AI stack built out over ~4 months in parallel. The core, tsumugi-ai, runs 9 CLI workflows (research, task decomposition, weekly review, health-log summarization, and more) on CrewAI + FastAPI + Ollama, backed by PostgreSQL (Alembic-managed) and OpenTelemetry distributed tracing. tsumugi-rag-teams is a standalone RAG service that syncs Microsoft Teams channel history via Graph API into Qdrant + sentence-transformers for vector search. tsumugi-desktop is a native macOS SwiftUI mascot app (99 Swift tests) that talks to the agent API over HTTP. tsumugi-governance enforces shared rules across every sub-repo through 5 layers — pre-commit, CI, hooks, runtime, and manual review.

  • Python (CrewAI/FastAPI) / Swift (SwiftUI) / Qdrant / OpenTelemetry / MS Graph API
  • Not just "using" agents — designing the system that governs multiple agents at once

tsumugi — 個人用マルチリポジトリAIエージェント基盤

約4ヶ月で並行構築した5リポジトリ構成のパーソナルAI基盤。中核の tsumugi-ai は CrewAI + FastAPI + Ollamaで9系統のCLI（調査・ToDo分解・週次レビュー・健康ログ要約など）を提供し、 Slack/Notion連携用にPostgreSQL（Alembic管理）とOpenTelemetryによる分散トレーシングまで実装。 tsumugi-rag-teams はMicrosoft Graph API経由でTeamsのチャネル履歴を同期し、 Qdrant + sentence-transformersでベクトル検索可能にする独立RAGサービス。 tsumugi-desktop はSwiftUI製のネイティブmacOSデスクトップマスコットで、 HTTP経由でエージェントAPIと対話する薄いUI層（Swiftテスト99件）。 tsumugi-governance は5段階の強制レイヤー（pre-commit/CI/hook/runtime/manual review）で ルールを機械検証可能にし、全サブリポジトリへ配布するガバナンス基盤。

AI GOVERNANCE EXPERIMENT

Designing a "constitution" for AI agents

This very page is served from that experiment. To let cheaper models operate with frontier-model discipline, I authored a 338-line, 22-chapter operating charter (decision priority, evidence discipline, stop conditions) and a 134-line, 12-section decision-heuristics reference, plus 8 least-privilege sub-agent definitions and a blinded model-comparison protocol.

AIエージェントの"憲法"設計

このページ自体が、その実験リポジトリから配信されている。安価なモデルでもフロンティアモデルと 同等の判断規律で自律稼働できるようにするため、決定優先順位・証拠規律・停止条件を明文化した ガイダンス文書（338行・22章）と、判断基準集（134行・12節）を設計。最小権限のサブエージェント 定義8種を実装し、モデル間のペア比較をブラインド化する比較手順も構築した。

Round-by-round scores in the working log are self-reported and not claimed here. This page states only what's independently verifiable — the documents, sub-agent definitions, and comparison protocol that actually exist in the repo.
